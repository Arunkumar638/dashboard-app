Variable Scope in Javascript

The scope of a variable is the part of the program from which the variable may directly be accessible.


In JavaScript, there are two types of scopes:


Global Scope

variables declared outside of any function or block have global scope. This means they can be accessed and modified anywhere in the code.

Here's an example:


// Global variable

var globalVariable = "I'm a global variable";

{"function sayGlobalVariable() {"}

  console.log(globalVariable); // Accessible here

{"}"}

sayGlobalVariable(); // Outputs: I'm a global variable

{"function modifyGlobalVariable() {"}

  globalVariable = "I've been modified";

{"}"}

modifyGlobalVariable();

sayGlobalVariable(); // Outputs: I've been modified

In this example, globalVariable is declared outside of any function, making it a global variable. It can be accessed both inside and outside of functions. The function sayGlobalVariable can access and print the value of globalVariable. The function modifyGlobalVariable can also modify the value of globalVariable, and those changes are reflected when we call sayGlobalVariable again.


Local Scope:

In JavaScript, variables declared within a function have local scope. This means they are only accessible within that specific function.

Here's an example:

function myFunction() {

  var localVar = "I'm a local variable"; // This is a local variable

  console.log(localVar); // Accessible here

{"}"}

myFunction(); // Outputs: I'm a local variable

console.log(localVar); // Error! localVar is not defined


In this example, localVar is declared within the function myFunction(). This makes it a local variable, which means it is only accessible within the context of that function. Trying to access localVar outside of myFunction() will result in an error, as demonstrated in the last line.

Local variables are useful for storing data that is specific to a particular function and doesn't need to be accessible elsewhere in the program. They also help prevent naming conflicts and improve code organization.

Keep in mind that if you declare a variable without using var, let, or const inside a function, it will become a global variable (or throw an error in strict mode). Therefore, it's a good practice to always use var, let, or const to declare variables, even within functions, to avoid unintentionally creating global variables.


JavaScript Let

"let" is a keyword in JavaScript used to declare variables with block scope. It was introduced in ECMAScript 6 (ES6) as an alternative to the older "var" keyword.

Here's an example of how you might use "let" in JavaScript:

let x = 10;

{"if (true) {"}

  let y = 20;

  console.log(x); // Output: 10

  console.log(y); // Output: 20

{"}"}

console.log(x); // Output: 10

console.log(y); // Error: y is not defined

In this example, x and y are both declared using "let". The key thing to notice is that y is only accessible within the block (the area between the curly braces) where it was declared. This is called block scope. Outside of that block, y is not accessible.

In contrast, if you had used var to declare y, it would have function scope, which means it would be accessible throughout the function, regardless of block scopes.

var x = 10;

{"if (true) {"}

  var y = 20;

  console.log(x); // Output: 10

  console.log(y); // Output: 20

{"}"}

console.log(x); // Output: 10

console.log(y); // Output: 20


JavaScript var

JavaScript var is a keyword used to declare variables in JavaScript that are function scoped. Before the introduction of ES6 all the keywords in JavaScript were declared with only “var” keyword. The var keyword is also used to declare global-scope variables.

Syntax:

var variableName = valueOfVar;

Function Scope:

The variables declared inside a function are function scoped and cannot be accessed outside the function. The variables declared with var can only be accessed inside that function and its enclosing function.

The variables declared using the var keyword are hoisted at the top and are initialized before the execution of code with a default value of undefined. The variables declared in the global scope that is outside any function cannot be deleted

JavaScript Const

ES2015 (ES6) introduced the const keyword to define a new variable. The difference in const variable declaration to others is that it cannot be re-assigned.

Properties:
  • Cannot be reassigned.
  • It’s Block Scope
  • It can be assigned to the variable on the declaration line.
  • It’s a Primitive value.
  • The property of a const object can be changed but it cannot be changed to a reference to the new object
  • The values inside the const array can be changed, it can add new items to const arrays but it cannot reference a new array.
  • Re-declaring of a const variable inside different block scopes is allowed.
  • Cannot be Hoisted.
  • Creates only read-only references to value.

    const const_name;

    const x;
