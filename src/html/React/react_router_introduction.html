Setting up react router

Setting up React Router allows you to implement client-side routing in your React application. This enables navigation between different pages or views without requiring a full page refresh.

Here are the steps to set up React Router:

Step 1: Install React Router

You can install React Router using npm or yarn. Open your terminal and run:

npm install react-router-dom

or

yarn add react-router-dom


Step 2: Create Router Component

In your application's entry point (usually index.js), you need to wrap your entire application with a BrowserRouter component. This component provides the routing context for your application.

import React from 'react';

import ReactDOM from 'react-dom';

import { BrowserRouter } from 'react-router-dom';

import App from './App'; // Your main component

ReactDOM.render(

  <BrowserRouter>

    <div>

  </BrowserRouter>,

  document.getElementById('root')

);


Step 3: Create Routes

In your main component (usually App.js), you'll define the routes for your application. Use the Route component to specify which component to render for each route.

import React from 'react';

import { Route, Switch } from 'react-router-dom';

import Home from './Home'; // Example component for the home page

import About from './About'; // Example component for the about page

import NotFound from './NotFound'; // Example component for 404 page

const App = () => {

  return (

    <div>

      <Switch>

        <Route exact path="/" component={Home} />

        <Route path="/about" component={About} />

        <Route component={NotFound} />

      </Switch>

    </div>

  );

};

export default App;


Step 4: Create Route Components

Now, you need to create the components for each route (Home, About, etc.). These components will be displayed when the corresponding route is matched.

// Home.js

import React from 'react';

const Home = () => {

  return <div>Home Page</div>;

};

export default Home;

// About.js

import React from 'react';

const About = () => {

  return <div>About Page</div>;

};

export default About;

// NotFound.js

import React from 'react';

const NotFound = () => {

  return <div>404 Page Not Found</div>;

};

export default NotFound;


Step 5: Linking Between Routes

To navigate between routes, use the Link component provided by React Router.

import React from 'react';

import { Link } from 'react-router-dom';

const Navigation = () => {

  return (

    <nav>

      <Link to="/">Home</Link>

      <Link to="/about">About</Link>

    </nav>

  );

};

export default Navigation;


Step 6: Start Development Server

Start your development server to see the React Router in action.

npm start

Your application should now be set up with React Router, allowing you to navigate between different routes without a full page refresh. Make sure to replace the example components (Home, About, etc.) with your actual components.


Understand routing in single page applications in react

In a Single Page Application (SPA) built with React, routing is the process of navigating between different views or pages within the application without causing a full page refresh. Instead of loading a new HTML page from the server, only the necessary components and data are updated in the DOM to reflect the new view.

React Router is a popular library used for managing routing in React applications. It provides a set of components and hooks that allow you to define routes and navigate between them.


Key Concepts in React Router:

BrowserRouter:

This component provides the routing context for your application. It uses the HTML5 history API to keep the UI in sync with the URL.

Example:

import { BrowserRouter } from 'react-router-dom';

<BrowserRouter>

  <App />

</BrowserRouter>


Route:

The Route component is used to declare a route and specify which component should be rendered when the URL matches that route.

Example:

import { Route } from 'react-router-dom';

<Route exact path="/" component={Home} />


Switch:

The Switch component is used to render the first child Route or Redirect that matches the current location. This ensures that only one route is rendered at a time.

Example:

import { Switch, Route } from 'react-router-dom';

<Switch>

  <Route exact path="/" component={Home} />

  <Route path="/about" component={About} />

  <Route component={NotFound} />

</Switch>


Link:

The Link component is used to create hyperlinks that navigate to different routes. It prevents the default behavior of refreshing the page.

Example:

import { Link } from 'react-router-dom';

<Link to="/about">About Us</Link>

NavLink:

Similar to Link, but allows for styling the active link (e.g., applying a class) when the route matches.

Example:

import { NavLink } from 'react-router-dom';

<NavLink to="/about" activeClassName="active">About Us</NavLink>


Redirect:

The Redirect component is used to redirect to a different route. It can be conditionally rendered based on some logic.

Example:

import { Redirect } from 'react-router-dom';

{loggedIn ? <Dashboard /> : <Redirect to="/login" />}

    c.Working with Browser Router and Hash Router components in react

    In React Router, BrowserRouter and HashRouter are two types of routers that you can use to handle routing in your application. They are used to synchronize the application's UI with the URL.


    BrowserRouter:

    BrowserRouter uses the HTML5 history API, specifically the pushState() and replaceState() methods, to keep the UI in sync with the URL. This allows for clean, semantic URLs without hash fragments.

    Here's an example of how to use BrowserRouter:

    import React from 'react';

    import { BrowserRouter, Route, Switch } from 'react-router-dom';

    import Home from './Home';

    import About from './About';

    import NotFound from './NotFound';

    const App = () => {

      return (

        <BrowserRouter>

          <Switch>

            <Route exact path="/" component={Home} />

            <Route path="/about" component={About} />

            <Route component={NotFound} />

          </Switch>

        </BrowserRouter>

      );

    };

    export default App;


    HashRouter:

    HashRouter uses the hash portion of the URL (the part after #) to keep the UI in sync with the URL. This is useful for environments that do not support the HTML5 history API, such as older browsers or static file servers.

    Here's an example of how to use HashRouter:

    import React from 'react';

    import { HashRouter, Route, Switch } from 'react-router-dom';

    import Home from './Home';

    import About from './About';

    import NotFound from './NotFound';

    const App = () => {

      return (

        </ashRouter>

          <Switch>

            <Route exact path="/" component={Home} />

            <Route path="/about" component={About} />

            <Route component={NotFound} />

        </HashRouter>

      );

    };

    export default App;


    When to Use BrowserRouter vs HashRouter:

    BrowserRouter:

    Use BrowserRouter when your server can handle dynamic, client-side routing. This means the server is configured to serve the same content for any path, and the client-side router (React Router) will handle rendering different components based on the URL.

    HashRouter:

    Use HashRouter when you have a static file server or need to support older browsers that do not handle HTML5 history API well. The HashRouter uses the hash portion of the URL, which is widely supported.

    Remember to also install the react-router-dom package if you haven't already:

    npm install react-router-dom

    Using BrowserRouter or HashRouter sets up the routing context for your application, allowing you to use other routing components like Route, Link, and Switch in your application.


    Configuring route with Route component in react

    In React Router, configuring routes is done using the Route component. The Route component allows you to specify which component should be rendered based on the current URL path.

    Here's how you can configure routes using the Route component:

    Step 1: Import Dependencies

    First, you need to import the necessary components from react-router-dom.

    import React from 'react';

    import { BrowserRouter, Route, Switch } from 'react-router-dom';

    import Home from './Home';

    import About from './About';

    import Contact from './Contact';

    import NotFound from './NotFound';

    Step 2: Set Up Routes

    Next, you'll define the routes and the corresponding components that should be rendered.

    const App = () => {

      return (

        <BrowserRouter>

          <Switch>

            <Route exact path="/" component={Home} />

            <Route path="/about" component={About} />

            <Route path="/contact" component={Contact} />

            <Route component={NotFound} />

          </Switch>

        </BrowserRouter>

      );

    };

    export default App;


    Explanation:

    BrowserRouter:

    Wrapping your entire application with BrowserRouter provides the routing context for your application.

    Switch:

    The Switch component ensures that only the first matching Route is rendered. It stops evaluating routes as soon as a match is found.

    Route:

    Each Route component is used to define a specific route and specify which component should be rendered when that route is matched.

    The exact prop ensures that the route matches exactly (and not partially).


    Creating Route Components:

    Each of the components (Home, About, Contact, NotFound) should be created in separate files (e.g., Home.js, About.js, etc.) and imported into the main file.

    Example Home.js:

    import React from 'react'

    const Home = () => {

      return  <div>Home Page </div>;

    };

    export default Home;


    Navigating Between Routes:

    To navigate between routes, you'll use the Link component provided by React Router.

    import React from 'react';

    import { Link } from 'react-router-dom';

    const Navigation = () => {

      return (

        <nav>

         <Link to="/">Home</Link>

         <Link to="/about">About</Link>

         <Link to="/contact">Contact</Link>

       </nav>

      );

    };

    export default Navigation;


    Description

    This Navigation component can be included in your Home, About, and Contact components to provide links for navigation.

    Remember to install the react-router-dom package if you haven't already:

    npm install react-router-dom

    With this setup, your application will now have three routes (/, /about, and /contact) that render different components based on the current URL path. If none of the routes match, the NotFound component will be rendered.
