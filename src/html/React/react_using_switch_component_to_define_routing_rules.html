Using Switch component to define routing rules in react

The Switch component in React Router is used to render the first Route or Redirect that matches the current location. It helps ensure that only one route is rendered at a time. If multiple routes match, only the first one will be rendered.


Here's an example of how to use the Switch component to define routing rules in React:

import React from 'react';

import { BrowserRouter as Router, Route, Switch } from 'react-router-dom';

import Home from './Home';

import About from './About';

import Contact from './Contact';

import NotFound from './NotFound';

const App = () => {

  return (

    <Router>

      <Switch>

        <Route exact path="/" component={Home} />

        <Route path="/about" component={About} />

        <Route path="/contact" component={Contact} />

        <Route component={NotFound} />

      </Switch>

    </Router>

  );

};

export default App;


In this example:

  • We import the necessary components from react-router-dom.
  • Inside the Switch component, we define multiple Route components with different paths and corresponding components to render.
  • The exact prop is used to ensure that a route matches exactly (and not partially).

  • The NotFound component is provided as a fallback route. If none of the specified routes match, this component will be rendered.

    • How Switch Works:

  • When a Switch is rendered, it searches through its children (Route or Redirect components).
  • It renders the first child Route or Redirect that matches the current location's path.
  • If no matching Route or Redirect is found, the Switch renders null (nothing).

    • Use Cases for Switch:

    404 Page:

    The Switch component is often used to define a 404 (Not Found) route that renders when no other routes match.

    Conditional Routing:

    You can use the Switch to conditionally render routes based on certain conditions in your application.

    Specific Order of Routes:

    If you have routes that have similar paths, you can use the Switch to ensure that the most specific route is matched first.

    Exclusive Routes:

    If you have routes that should not be rendered together (e.g., login and register pages), you can use the Switch to ensure only one is rendered at a time.

    The Switch component is a powerful tool for controlling the flow of your application based on the current URL path. It ensures that only one route is rendered at a time, providing a smooth and predictable navigation experience.
