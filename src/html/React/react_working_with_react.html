Working with React.

Create Element

In React, you can create elements using either JSX or the React.createElement function. Using React.createElement is less common than using JSX, but it's important to understand because it's the underlying method that JSX gets transpiled into.


Here's an example of how to create an element using React.createElement:


import React from 'react';

import ReactDOM from 'react-dom';

const element = React.createElement('h1', null, 'Hello, World!');

ReactDOM.render(element, document.getElementById('root'));


In this example, we're creating a React element of type h1 with the text content "Hello, World!". The React.createElement function takes three arguments:


  1. The type of element you want to create (e.g., 'div', 'h1', 'span', etc.).
  2. An object for the element's properties or attributes. We're passing null here, but you could pass an object with properties like className, style, etc.
  3. The content of the element. This can be a string, another element, or an array of elements.
  4. The React.createElement function returns a JavaScript object representing the element. It doesn't actually create any DOM nodes. To render it to the actual DOM, you use ReactDOM.render.

In this example, we're rendering the element to an element with the ID of 'root'. You should have an HTML file with a <div id="root"></div> for this to work. Again, it's worth noting that while understanding React.createElement is important for understanding how React works under the hood, in practice, most React developers use JSX as it provides a more concise and readable way to define UI components.
