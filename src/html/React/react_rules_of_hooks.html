Rules of hooks

Here are some important rules and guidelines for using hooks in React:


1.    Only Call Hooks at the Top Level:

Don't call hooks inside loops, conditions, or nested functions. Always use them at the top level of your functional component or custom hook.


// Correct

function MyComponent() {

  const [count, setCount] = useState(0);

  // ...

}

// Incorrect

function MyComponent() {

  if (condition) {

    const [count, setCount] = useState(0); // Incorrect

  }

  // ...

}


2.    Only Call Hooks from React Functions:

You can call hooks from React functional components or custom hooks. Don't call them from regular JavaScript functions.

// Correct

function MyComponent() {

  const [count, setCount] = useState(0);

  // ...

}

// Incorrect

const myFunction = () => {

  const [count, setCount] = useState(0); // Incorrect

  // ...

}


3.    Only Call Hooks from Custom Hooks or React Functions:

If you're creating a custom hook, make sure to call other hooks from it, and don't call hooks from regular JavaScript functions.

// Correct

function useCustomHook() {

  const [value, setValue] = useState(0);

  // ...

}

// Incorrect

const myFunction = () => {

  const [count, setCount] = useState(0); // Incorrect

  // ...

}


4.    Use Hooks in the Same Order:

Always use hooks in the same order within your component. This helps React to maintain the correct state between renders.

// Correct

function MyComponent() {

  const [count, setCount] = useState(0);

  const [text, setText] = useState('');

  // ...

}

// Incorrect

function MyComponent() {

  const [text, setText] = useState('');

  const [count, setCount] = useState(0); // Incorrect order

  // ...

}


5.    Only Call Hooks Conditionally (if required):

Hooks should not be called conditionally, meaning they should always be called in the same order on every render. If you need to conditionally use a hook, you can do so in a custom hook.

// Correct

function MyComponent() {

  if (condition) {

    const [count, setCount] = useState(0);

    // ...

  }

  // ...

}

// Correct

function useConditionalHook() {

  if (condition) {

    const [count, setCount] = useState(0);

    // ...

  }

  // ...

}


6.    Only Use Hooks in Functional Components:

Hooks can only be used in functional components or custom hooks. They cannot be used in class components.

// Correct

function MyComponent() {

  const [count, setCount] = useState(0);

  // ...

}

// Incorrect

class MyClassComponent extends React.Component {

  render() {

    const [count, setCount] = useState(0); // Incorrect

    // ...

  }

}


7.    Don't Call Hooks from Regular JavaScript Functions:

Hooks should only be called from within functional components or custom hooks, not from regular JavaScript functions.

// Correct

function useCustomHook() {

  const [value, setValue] = useState(0);

  // ...

}

// Incorrect

function myFunction() {

  const [count, setCount] = useState(0); // Incorrect

  // ...

}


8.    Custom Hooks Must Start with "use":

If you create a custom hook, it must always start with the word "use". This convention is how React recognizes it as a hook.

// Correct

function useCustomHook() {

  // ...

}

// Incorrect

function customHook() {

  // ...

}

Following these rules will help you use hooks effectively and avoid common pitfalls. Keep in mind that adherence to these rules is important to ensure that your components work correctly and maintain the intended behavior.
