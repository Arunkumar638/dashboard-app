e.Suspense

Suspense is a React component that allows you to declaratively specify loading states while components are dynamically loading. It's primarily used in conjunction with React.lazy for code splitting and data fetching.


Here's how Suspense works:

  • It's a component that you wrap around code that may throw a promise, such as a dynamic import or a data fetching function.
  • While the promise is pending, the nearest Suspense ancestor will pause rendering and display a fallback UI (specified by the fallback prop).
  • Once the promise resolves or rejects, the rendering continues as usual.

    • Here's an example of how to use Suspense with React.lazy:

    import React, { Suspense } from 'react';

    const LazyComponent = React.lazy(() => import('./LazyComponent'));

    const App = () => {

      return (

        <div>

          <Suspense fallback={<div>Loading...</div>}>

            <LazyComponent />

          </Suspense>

        </div>

      );

    };

    export default App;


    In this example:

    <Suspense> is used to wrap the component that might throw a promise (in this case, the LazyComponent).

    The fallback prop specifies the fallback UI to display while the LazyComponent is loading.


    Using Suspense for Data Fetching:

    Suspense can also be used for data fetching, although it's an experimental feature.

    import React, { Suspense } from 'react';

    import { fetchUserData } from './api'; // Example data fetching function

    const UserData = React.lazy(() => fetchUserData());

    const App = () => {

      return (

        <div>

          <Suspense fallback={<div>Loading user data...</div>}>

            <UserData />

          </Suspense>

        </div>

      );

    };

    export default App;


    In this example, UserData is a component that is dynamically imported using React.lazy, and it might throw a promise due to data fetching.

    If the data fetching takes some time, the Suspense component will display the fallback UI ("Loading user data...") until the data is ready.


    Suspense is a powerful tool for managing loading states in your React application, whether for code splitting or data fetching. It allows you to declaratively handle loading states, making your code more readable and maintainable.
