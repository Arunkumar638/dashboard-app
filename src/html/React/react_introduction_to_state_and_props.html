State and props

State and props are two fundamental concepts in React that allow you to manage data and pass information between components.

State:

State is an internal data store that belongs to a specific component. It represents the current condition or data of a component at any given point in time. State can be modified by the component itself, typically in response to user actions or other events.

Setting Initial State:

You initialize state in a class component by setting the this.state property in the component's constructor.


class Counter extends React.Component {

constructor(props) {

super(props);

this.state = {

count: 0

};

}

render() {

return <div>Count: {this.state.count}</div>;

}

}


Updating State:

You should never directly modify the state object. Instead, you use the setState method provided by React. This ensures that React knows about the update and can trigger a re-render.
this.setState({ count: this.state.count + 1 });


Asynchronous Nature of setState:

The setState function is asynchronous, which means that React may batch multiple setState calls into a single update for performance reasons. If you need to perform actions after the state has been updated, you can use the callback function as the second argument to setState.


this.setState({ count: this.state.count + 1 }, () => {

console.log('Count updated:', this.state.count);

});


Props:

Props (short for properties) are inputs that are passed into a React component. They are immutable (read-only) within the component that receives them. Props allow you to customize and configure a component when it is used in a parent component.


Passing Props:

You pass props in the JSX element when you use a component.

<ChildComponent name="John Doe" age={30} />

In this example, name and age are props being passed to the ChildComponent.


Accessing Props:

In the child component, you can access props via this.props.


class ChildComponent extends React.Component {

render() {

return (

<div>

<p>Name: {this.props.name}</p>

<p>Age: {this.props.age}</p>

</div>

);

}

}

Here, this.props.name and this.props.age are accessing the props passed from the parent.


Default Props:

You can provide default values for props using the defaultProps property.


class ChildComponent extends React.Component {

static defaultProps = {

name: 'Unknown',

age: 0

};

render() {

return(

<div>

<p>Name: {this.props.name}</p>

<p>Age: {this.props.age}</p>

</div>

);

}

}

If a prop is not provided when using the component, it will use the default value.
