Pointer arithmetic

Pointer arithmetic is a feature in C that allows you to perform arithmetic operations on pointers. This is possible because pointers are just memory addresses, and their values are typically integers representing locations in memory.
Here are some important things to know about pointer arithmetic:
Increment and Decrement:
You can increment and decrement a pointer by an integer value. The size of the increment or decrement is based on the data type the pointer is pointing to.
int arr[5] = {1, 2, 3, 4, 5};
int *ptr = arr; // ptr points to the first element of arr
ptr++; // Moves ptr to the next integer location in memory
Addition and Subtraction:
You can add or subtract an integer from a pointer. Again, the size of the increment or decrement is based on the data type.
int arr[5] = {1, 2, 3, 4, 5};
int *ptr = arr; // ptr points to the first element of arr
ptr = ptr + 2; // Moves ptr two elements ahead (8 bytes for integers)
Array Indexing:
Array indexing can be viewed as a form of pointer arithmetic. The expression arr[i] is equivalent to *(arr + i), where arr is a pointer to the first element of the array.
int arr[5] = {1, 2, 3, 4, 5};
int *ptr = arr; // ptr points to the first element of arr
printf("%d\n", *(ptr + 2)); // This will print the value 3
Pointer Differences:
You can subtract one pointer from another, as long as they are both pointers to the same data type. The result will be the number of elements between them.
int arr[5] = {1, 2, 3, 4, 5};
int *ptr1 = arr; // ptr1 points to the first element of arr
int *ptr2 = arr + 3; // ptr2 points to the fourth element of arr
int diff = ptr2 - ptr1; // diff will be 3, because there are 3 elements between ptr1 and ptr2
Void Pointers and Pointer Arithmetic:
Void pointers (void*) cannot be directly dereferenced, but they can be typecasted to other pointer types to perform pointer arithmetic. Be cautious, as typecasting a void pointer incorrectly can lead to undefined behavior.
Pointer arithmetic is a powerful feature, but it also requires caution. You must ensure that you are performing operations on valid memory locations to avoid undefined behavior or memory access violations. Additionally, it's important to be aware of the size of the data type being pointed to, as this affects the behavior of pointer arithmetic.
