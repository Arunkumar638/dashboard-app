Dynamic memory allocation malloc

Dynamic memory allocation is a feature in C that allows you to allocate memory at runtime. This is particularly useful when you don't know the size of the data you need to store beforehand, or if you need to dynamically resize data structures.
The malloc() function is used for dynamic memory allocation in C. It stands for "memory allocation." Here's how you use it:
#include <stdio.h>
#include <stdlib.h>
int main() {
int *ptr;
// Allocate memory for one integer
ptr = (int *)malloc(sizeof(int));
if (ptr == NULL) {
printf("Memory allocation failed\n");
return 1; // Indicate an error
}
// Use the allocated memory

*ptr = 42;
printf("Value: %d\n", *ptr);
// Free the allocated memory when done
free(ptr);
return 0;
}
Explanation:
#include <stdlib.h>
: This header file provides the malloc() and free() functions.
int *ptr;: Declares a pointer ptr to hold the address of the dynamically allocated memory.
ptr = (int *)malloc(sizeof(int));: Allocates memory for one integer using malloc(). The sizeof(int) is used to determine the size of the integer in bytes. malloc() returns a pointer to the allocated memory, which is then typecasted to an int*.
if (ptr == NULL) { ... }: Checks if malloc() was successful. If it returns NULL, it means that the allocation failed (perhaps due to insufficient memory).
*ptr = 42;: Assigns a value to the allocated memory.
printf("Value: %d\n", *ptr);: Prints the value stored in the allocated memory.
free(ptr);: Releases the dynamically allocated memory. This is crucial to avoid memory leaks.
Remember:
Always check if malloc() returns NULL to handle the case where memory allocation fails. After allocating memory with malloc(), you should always free it using free() when you're done with it. This helps prevent memory leaks. Dynamic memory allocation is powerful, but it also requires you to manage memory manually, which can lead to errors if not done carefully. It's important to free the memory when you're done using it to avoid memory leaks.
